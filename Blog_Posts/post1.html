Problem 1:

In this problem, I start off by creating a list called eligible_numbers which will basically collect all the numbers that meet our condition and put them in the list. I then create a for-loop to search through between 2000 and 3201 . 3201 because I want to make sure that 3200 is counted in the range. If I made it from 2000 to 3200, then the for-loop would stop after 2900. I then have an if-condition checking for our required condition. I'll append every value between this range that meets our condition to the eligible_numbers list.

After this loop, I create another one to try to print all the eligible numbers. I append them to a variable a, and I use string formatting techniques to get rid of the last space and comma. This way, I have a good output.

If the user enters a string value, or anything that isn't an integer, a ValueError will rise, causing the program to go to the except statement and automatically quitting. If the user types in a negative value, the program will quit. Anything else will work.

I set the initial value of the answer to 1 and I multiply answer by the check value, which is set to the integer value of the user-input. This way, as the check value decrements by 1, it keeps getting multiplied into the final answer. Sure enough, if you enter 8, you'll get 40320 as given in the original example.
